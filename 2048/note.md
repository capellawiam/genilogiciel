# TP – Retour d’expérience GitHub & GitHub Copilot

Au début de ce TP, mon environnement de travail n’était pas correctement configuré :  
mon VS Code n’était pas relié à mon compte GitHub, et sur GitHub, GitHub Copilot était uniquement disponible en version gratuite.

À la suite de ce TP, j’ai effectué une demande pour obtenir le package **GitHub Copilot Pro**, ce qui m’a permis d’utiliser pleinement les fonctionnalités de l’intelligence artificielle.  
J’ai également appris à connecter VS Code à mon compte GitHub et à gérer un projet directement depuis l’interface de VS Code, sans passer systématiquement par le terminal (commit, push, synchronisation).

Ce TP m’a permis de comprendre qu’il est possible de **créer une application complète uniquement avec GitHub Copilot**, en utilisant l’IA à la fois pour :
- écrire le code,
- structurer le projet,
- déboguer les erreurs,
- améliorer progressivement l’application.

Au début, cela a été assez difficile, notamment pour comprendre :
- comment configurer GitHub Copilot Pro,
- comment lier VS Code à GitHub,
- comment utiliser Git correctement (commit, push),
- et comment exploiter efficacement l’intelligence artificielle.

Avant ce TP, je pensais que GitHub servait uniquement à héberger du code ou à créer un portfolio.  
Après ce TP, j’ai compris que GitHub offre de **nombreux avantages aux étudiants**, notamment grâce à ses outils collaboratifs et à l’intégration de l’intelligence artificielle.

Ce TP m’a permis de réaliser que l’IA est un **véritable outil de développement**, à condition de savoir comment l’utiliser correctement.
